The ACME LUCK Protocol
draft-sheffer-acme-luck-latest

Abstract

This document describes ACME LUCK (Limited Use of Cached Keys), a protocol that allows a domain owner to delegate some other entity (which we will call “the TLS server”) the authority to terminate TLS connections on its behalf, using short-term certificates. These certificates, whose validity is measured in days (e.g., 3 days), allow the domain owner to withdraw the TLS server’s authorization when necessary, without requiring certificate revocation. In order to attain a high degree of automation for the certificate generation and provisioning processes, this protocol extends the ACME interfaces.
